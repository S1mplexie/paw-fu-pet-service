## 运行测试

### 方式1：使用提供的脚本

**Windows系统：**
```bash
双击运行 run-tests.bat
```

**Linux/Mac系统：**
```bash
chmod +x run-tests.sh
./run-tests.sh
```

### 方式2：使用Maven命令

```bash
# 运行所有测试
mvn clean test

# 运行指定测试类
mvn test -Dtest=UserControllerTest

# 运行指定包下的测试
mvn test -Dtest="com.example.petadoption.controller.*"

# 生成测试覆盖率报告
mvn clean test jacoco:report
```

### 方式3：在IDE中运行

**IntelliJ IDEA：**
1. 右键点击 `src/test/java` 目录
2. 选择 "Run 'All Tests'"
3. 或右键单个测试类/方法运行

**Eclipse：**
1. 右键点击项目
2. 选择 "Run As" -> "JUnit Test"

## 测试技术栈

- **JUnit 5 (Jupiter)** - 测试框架
- **Mockito** - Mock框架
- **Spring Boot Test** - Spring测试支持
- **H2 Database** - 内存数据库
- **JaCoCo** - 覆盖率工具

## 测试特性

### 1. 单元测试独立性
- 每个测试方法独立运行
- 使用 `@BeforeEach` 初始化测试环境
- 测试之间互不影响

### 2. Mock模拟
- 使用 `@Mock` 模拟依赖对象
- 使用 `@InjectMocks` 注入测试对象
- 避免真实数据库和网络调用

### 3. 异常测试
- 使用 `assertThrows` 验证异常
- 覆盖所有业务异常场景

### 4. 边界条件
- 空值测试
- 超长字符串测试
- 特殊字符测试
- 格式校验测试

## 注意事项

1. **数据库配置**
   - 测试使用H2内存数据库
   - 不会影响真实数据库
   - 测试完成后数据自动清除

2. **JWT配置**
   - 测试环境使用独立的密钥
   - 不会影响生产环境配置

3. **Redis配置**
   - 测试环境可使用嵌入式Redis
   - 或使用Mock模拟Redis操作

## 常见问题

### Q1: 测试失败如何调试？
A: 查看测试报告中的错误堆栈，或在IDE中以调试模式运行测试。

### Q2: 如何增加新测试？
A: 参考现有测试类的编写方式，使用相同的注解和Mock方式。

### Q3: 覆盖率不达标怎么办？
A: 查看JaCoCo报告，找出未覆盖的代码，补充相应的测试用例。

### Q4: 测试运行很慢怎么办？
A: 可以单独运行某个测试类，或使用Maven并行测试选项。

## 持续集成

可以将测试集成到CI/CD流程：

```yaml
# GitHub Actions示例
- name: Run Tests
  run: mvn clean test

- name: Generate Coverage Report
  run: mvn jacoco:report

- name: Upload Coverage to Codecov
  uses: codecov/codecov-action@v2
  with:
    file: ./target/site/jacoco/jacoco.xml
```
